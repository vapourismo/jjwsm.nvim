==============================================================================
REQUIREMENTS                                               *jjwsm-requirements*

Neovim 0.10+, Jujutsu 0.40+, and Snacks.nvim with its picker enabled are
required. Tabby.nvim is optional and names successfully activated workspace
tabs. The plugin registers its command automatically; do not call setup().

==============================================================================
COMMANDS                                                       *jjwsm-commands*

                                                                     *:Jjwsm*
                                                               *:Jjwsm-switch*
:Jjwsm switch
    Open a Snacks picker containing the other live workspaces registered in
    the invoking tabpage cwd's Jujutsu repository. Workspace names and roots
    are both searchable. Confirmation applies :tcd only to the tabpage that
    invoked the command.

                                                                  *:Jjwsm-new*
:Jjwsm new
    Prompt for a workspace name, create it under the shared temporary parent,
    and open it in a blank tabpage with a tab-local cwd. The prompt uses
    vim.ui.input() so UI providers can customize it.

                                                               *:Jjwsm-delete*
:Jjwsm delete
    Close the invoking tabpage and forget its workspace from Jujutsu metadata.
    This command never removes the workspace directory or any other filesystem
    content.

The command itself is global because user commands cannot be tab-local. The
invoking tabpage cwd is captured and validated on every invocation.

==============================================================================
TEMPORARY WORKSPACES                                         *jjwsm-temporary*

The workspace identifier comes from a `Workspace name: ` prompt with no
generated default. Nonblank input is preserved verbatim. Cancelling reports an
informational notice. Blank names and names already registered in the current
repository are rejected.

The temporary path is generated independently from the identifier. Its exact
layout is: >

    $TMPDIR/jjwsm.nvim/jjwsm-<repository>-N
<
For a default workspace root of /work/my-repo, the first generated path is: >

    $TMPDIR/jjwsm.nvim/jjwsm-my-repo-1
<
The temporary root is obtained from vim.uv.os_tmpdir(). The jjwsm.nvim parent
is created with mode 0700 when absent. An existing parent must be a real
directory. Allocation begins at 1 and skips every generated path that already
exists. A concurrent path collision advances to the next counter without
changing the prompted name or repository basename. If a collision rescan finds
that the prompted name was concurrently registered, creation stops.

The final directory is initially absent; `jj workspace add` creates it. No PID,
timestamp, or other component is appended to the generated path basename. The
repository basename is copied verbatim from the retained default workspace
root, preserving spaces, case, dots, and punctuation. If that workspace, its
root, or a non-empty basename cannot be determined, creation stops before the
prompt is displayed or the temporary parent is created. Cancellation, blank
input, and duplicate names also stop before the parent is created. Other name
validation is left to Jujutsu and its failure is reported.

==============================================================================
DELETING WORKSPACES                                           *jjwsm-delete*

Delete resolves the most-specific registered workspace containing the
invoking tabpage cwd. The default workspace cannot be deleted, and deletion is
also refused if only one tabpage exists. The latter restriction is checked
both before starting a process and immediately before closing the tabpage.

The captured tabpage must still exist and resolve to the same workspace after
the asynchronous workspace lookup. It is closed first with a targeted,
non-forced :tabclose, allowing Neovim to protect modified buffers. If Neovim
refuses the close, the workspace stays registered. If the close succeeds but
`jj workspace forget` fails, the error is reported without recreating the tab.

Only the Jujutsu workspace record is forgotten. The workspace directory and
all of its contents remain unchanged. Other tabpages are untouched, including
tabs that refer to the same workspace.

==============================================================================
TABBY INTEGRATION                                               *jjwsm-tabby*

After a successful non-default switch or workspace creation, jjwsm.nvim
invokes: >

    :Tabby rename_tab <repository>[<workspace>]
<
<repository> is the retained default workspace root's basename and <workspace>
is the selected or prompted workspace name. Both are copied verbatim, including
spaces, case, dots, and punctuation. For a default root of
/work/Repo With Spaces.v1+Draft and a prompted name of feature: parser work,
the new tab is named: >

    Repo With Spaces.v1+Draft[feature: parser work]
<
Switching updates the tabpage that invoked the picker, even if another tabpage
is current at confirmation time. Switching to the default workspace invokes: >

    :Tabby rename_tab
<
This removes that tab's previous explicit Tabby name. Other successful
activations replace any prior Tabby name on that tab.

Naming is best-effort. If :Tabby is unavailable, renaming fails, or the default
basename cannot be determined during a non-default switch, jjwsm.nvim warns but
preserves the activated cwd and previous tab name. Failed and stale activations
do not update tab names.

==============================================================================
CLEANUP AND SAFETY                                               *jjwsm-cleanup*

Before switch or new, jjwsm.nvim inspects all workspaces registered in the
current repository. If legacy metadata gives the current workspace no recorded
path, its actual root is recovered transparently with `jj workspace root`
without rewriting the workspace record. Missing and non-directory roots are
forgotten from Jujutsu. Live directories are retained, including workspaces
created by other tools. Indeterminate roots (such as permission failures) are
retained with a warning. A failure from `jj workspace forget` aborts the
requested operation.

Delete still recovers the current root, but lists workspaces without this
stale-record cleanup and forgets only its resolved current workspace after the
tabpage closes successfully.

The plugin never removes a directory or file. Existing candidate directories
are never reused, including empty directories. Temporary workspaces persist
until removed by the user or operating system.
